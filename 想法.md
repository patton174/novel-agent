# 写小说 AI Agent 设计方案（完善版）

## 一、整体架构概览

```
[ 用户浏览器 / 移动端 ]
           │
           ▼
[ Web 前端 (React 18 + Next.js) ] ──── WebSocket（实时进度）
           │ HTTP/REST
           ▼
[ Java 后端 (Spring Boot 3) ] ──────── [ PostgreSQL + Redis ]
           │                               │
           │ REST / gRPC                    │
           ▼                               │
[ Python AI 服务 (FastAPI) ] ◄────────── [ Milvus 向量库 ]
           │                              │
           │ 调用 LLM (DeepSeek / GPT-4o)  │
           │ 使用 LangChain 编排 Agent     │
           │                    │
           └── RabbitMQ 异步队列 ◄────────┘
```

### 架构分层说明

| 层级 | 组件 | 职责 |
|------|------|------|
| 前端 | React 18 + Next.js | 小说编辑器、AI 助手面板、项目管理 |
| Java 后端 | Spring Boot 3 + Spring Security | 账号体系、权限控制、API 网关、任务调度、持久化 |
| AI 服务 | FastAPI + LangChain | LLM 调用、Agent 逻辑、文本处理、向量检索 |
| 消息队列 | RabbitMQ | 长文本生成等异步任务的削峰填谷 |
| 数据库 | PostgreSQL + Redis | 结构化数据存储、会话缓存、分布式锁 |
| 向量库 | Milvus | 角色设定、世界观、历史片段的语义检索 |

### 关键设计原则

1. **Java 不直接调用 LLM**：业务逻辑与 AI 能力分离，Java 只做请求转发
2. **Python 不持有业务数据**：只处理文本，不存储用户信息
3. **密钥各自独立管理**：Java 和 Python 分别从环境变量读取 API 密钥
4. **向量库由 Java 统一管理连接**：Python 通过 Java 提供的接口访问向量库

## 二、技术栈选型（2026 年推荐）

| 层次 | 推荐选型 | 备选方案 | 说明 |
|------|---------|---------|------|
| Java 框架 | Spring Boot 3 + Spring Security 6 | Spring Boot 2 + Shiro | JWT/OAuth2 认证，稳定生态 |
| Python 框架 | FastAPI + LangChain | FastAPI + LlamaIndex | 高性能异步，LangChain 工具链更成熟 |
| LLM | DeepSeek-V3（性价比） / GPT-4o（质量） | Claude 3.5 / Qwen3 | 支持多模型切换，按需选择 |
| 向量库 | Milvus（生产级） | Qdrant / Chroma | 支持分布式，运维成熟 |
| 数据库 | PostgreSQL 16 | MySQL 8 | JSON 支持好，适合半结构化数据 |
| 缓存 | Redis 7 | - | 会话、分布式锁、草稿缓存 |
| 消息队列 | RabbitMQ 3.13 | Kafka | 运维简单，满足当前规模 |
| 前端 | React 18 + Next.js 14 | Vue 3 + Nuxt | SSR 支持好，生态丰富 |
| 部署 | Docker Compose | Kubernetes | 单机用 Compose，多机用 K8s |

**选型说明**：
- LangChain 与 LlamaIndex：LangChain 工具链更完善，LlamaIndex 在文档检索场景性能更好。本项目以生成为主，选 LangChain。
- RabbitMQ 与 Kafka：当前规模 RabbitMQ 足够，运维简单。等消息量超过 10 万/日再迁移 Kafka。

## 三、模块详细设计

### 3.1 Java 后端核心模块

#### 用户与认证

- 注册 / 登录：JWT + Refresh Token
- 密码存储：bcrypt（强度 12）
- 权限体系：普通用户 / VIP / 管理员
- 会话管理：Redis 存储，TTL 7 天

#### 项目管理

```
用户 ──→ 小说项目 ──→ 卷 ──→ 章节 ──→ 版本历史
                      │
                      ├─→ 角色卡
                      └─→ 世界观设定
```

- 小说类型：都市 / 玄幻 / 科幻 / 悬疑 / 古言
- 风格标签：爽文 / 文青 / 严谨 / 轻松
- 支持分卷，每卷可独立设置时代背景

#### 写作交互 API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/novels/{id}/chapters/{cid}/continue` | POST | 续写当前章节 |
| `/api/novels/{id}/chapters/{cid}/rewrite` | POST | 重写指定段落 |
| `/api/novels/{id}/outline/generate` | POST | 生成小说大纲 |
| `/api/novels/{id}/chapters/{cid}/dialogue` | POST | 生成角色对话 |
| `/api/novels/{id}/chapters/{cid}/review` | POST | AI 审校 |
| `/api/drafts/{id}/adopt` | POST | 采用草稿 |
| `/api/drafts/{id}/discard` | DELETE | 丢弃草稿 |

#### 异步任务处理

长文本生成（单章节 > 2000 字）走 RabbitMQ 异步：

1. 前端请求 → Java 创建任务，返回 `taskId`
2. Java 将任务投递到队列
3. Python 消费任务，处理完成后回调 Java
4. 前端通过 WebSocket 订阅任务进度

#### 草稿管理

```sql
CREATE TABLE drafts (
    id BIGINT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    chapter_id BIGINT NOT NULL,
    content TEXT NOT NULL,
    candidates INT DEFAULT 1,
    status VARCHAR(20),
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);
```

### 3.2 Python AI 服务核心模块

#### Agent 内部结构

```
输入（前文 + 用户指令）
         │
         ▼
[ 上下文组装器 ] → 从 Milvus 检索相关角色、设定、伏笔
         │
         ▼
[ Prompt 构造器 ] → 根据任务类型选择模板
         │
         ▼
[ LLM 调用 ] → DeepSeek-V3 / GPT-4o
         │
         ▼
[ 后处理器 ] → 格式清理、敏感词过滤、字数控制
         │
         ▼
返回 Java
```

#### 工具与记忆

| 工具 | 说明 |
|------|------|
| `vector_search` | 检索角色设定、世界观、历史片段 |
| `character_state` | 维护角色当前状态（位置、心情、携带物品） |
| `plot_tracking` | 跟踪伏笔，记录已埋设的线索 |
| `summary_memory` | 章节摘要，超过上下文长度时注入 |

#### Prompt 模板

| 任务 | 模板说明 |
|------|---------|
| 续写 | 前文 + 风格要求 → 生成 3 个候选 |
| 重写 | 原段落 + 修改要求 → 生成 3 个候选 |
| 大纲生成 | 一句话梗概 → 三幕结构 + 章纲 |
| 对话生成 | 角色A + 角色B + 场景 → 对话文本 |
| 审校 | 正文 → 问题列表 + 修改建议 |

#### 后处理

- 格式清理：移除多余空行、统一引号样式
- 敏感词过滤：关键词库匹配 + 正则替换
- 字数控制：误差 ±5%

## 四、错误处理与容错

### 4.1 LLM 调用失败

| 场景 | 处理策略 |
|------|---------|
| 超时（> 30s） | 重试 2 次，间隔 5s、15s |
| 返回错误码 | 重试 1 次，间隔 3s |
| 连续失败 3 次 | 返回友好错误："AI 服务暂时繁忙，请稍后再试" |

### 4.2 向量库故障

| 场景 | 处理策略 |
|------|---------|
| 连接超时 | 降级：跳过向量检索，纯前文生成 |
| 连接失败 | 告警 + 降级，同上 |
| 检索无结果 | 返回空，继续生成 |

### 4.3 数据库故障

| 场景 | 处理策略 |
|------|---------|
| 写入失败 | 事务回滚，返回错误 |
| 读取失败 | 返回缓存（如果有）或报错 |

### 4.4 网络故障

- Java 调用 Python 超时：返回 504，前端显示重试按钮
- 前端断网：本地草稿暂存，恢复后自动同步

### 4.5 异步任务失败

```
任务失败 → 死信队列（DLQ）→ 告警 → 人工处理
```

RabbitMQ 配置：
- 重试次数：3
- 重试间隔：指数退避（1min, 5min, 30min）
- 死信交换机：`novel.ai.dlx`

## 五、安全性设计

### 5.1 认证与授权

- JWT Access Token：有效期 1 小时，存用户 ID、角色
- Refresh Token：有效期 7 天，存 Redis
- 权限控制：RBAC（角色-权限映射）

### 5.2 API 密钥管理

```
Java: 环境变量 OPENAI_API_KEY → Spring @Value → RestTemplate
Python: 环境变量 DEEPSEEK_API_KEY → os.getenv → langchain
```

- 各自服务独立管理，不跨服务传递
- API 密钥不在日志中打印（脱敏）
- 定期轮转（每 90 天）

### 5.3 数据安全

- 密码：bcrypt 加密，不可逆
- 小说内容：用户私有，API 层校验 ownership
- 敏感操作日志：记录到审计表

### 5.4 SQL 注入防护

- 使用 Spring Data JPA 的参数化查询
- 禁止拼接 SQL 字符串
- 敏感字段过滤

## 六、限流与熔断

### 6.1 API 限流

| 维度 | 限制 | 说明 |
|------|------|------|
| 用户 | 60 请求/分钟 | 防止滥用 |
| VIP 用户 | 300 请求/分钟 | 付费特权 |
| IP | 100 请求/分钟 | 防止 DDoS |

实现：Redis + 滑动窗口算法

### 6.2 LLM 熔断

```
正常：请求 → LLM → 响应
熔断触发条件：
  - 5 分钟内错误率 > 30%
  - 平均响应时间 > 20s

熔断后：
  - 降级返回："AI 服务繁忙，请稍后重试"
  - 30 秒后尝试恢复
```

### 6.3 成本控制

| 措施 | 说明 |
|------|------|
| 单次最大 Token | 4096 |
| 月度额度警告 | 80% 时邮件通知 |
| 超额自动限流 | 95% 时降为 10 请求/分钟 |

## 七、内容审核

### 7.1 敏感词过滤（第一级）

- 关键词库：政治、色情、暴力等敏感词
- 实现：Trie 树 + 正则匹配
- 匹配到 → 直接拒绝，提示"内容包含敏感词"

### 7.2 AI 辅助审核（第二级）

- 正常返回内容抽样 10% 送审
- 审核维度：政治倾向、色情程度、暴力描述
- 高风险内容 → 下线处理，通知用户

## 十二、关键交互流程

### 12.1 用户点击"续写"

```
1. 前端 POST /api/novels/{id}/chapters/{cid}/continue
2. Java 鉴权、限流检查
3. Java 存储当前正文到 drafts 表（status=pending）
4. Java 同步调用 Python POST /ai/continue
5. Python 执行：
   a. 从向量库检索相关设定（前3章相关角色）
   b. 组装 Prompt（续写模板 + 前文 + 检索结果）
   c. 调用 DeepSeek API
   d. 后处理（格式、敏感词）
6. Python 返回生成结果（3个候选）
7. Java 更新 drafts 表，存储3个候选
8. 返回前端展示
9. 用户选择候选，点击"采用"
10. Java 将正文更新到 chapters 表，保存版本历史
```

### 12.2 异步长文本生成

```
1. 前端 POST /api/novels/{id}/chapters/{cid}/generate（body: {async: true}）
2. Java 创建任务（taskId），投递给 RabbitMQ
3. Java 立即返回 taskId 给前端
4. Python 消费队列，开始生成
5. Python 通过 WebSocket 推送进度（10%、30%、50%...）
6. 生成完成后，Python 回调 Java（POST /api/tasks/{id}/callback）
7. Java 更新 drafts 表
8. 前端收到完成通知，展示结果
```

## 十四、进阶功能展望

> 以下功能在 MVP 阶段不实现，记录在此指导架构扩展

1. **分层规划**：小说梗概 → 章纲 → 细纲 → 逐段写作，每步可人工修正
2. **角色状态跟踪**：维护"当前状态表"（位置、心情、物品），避免前后矛盾
3. **伏笔管理**：标记伏笔，AI 后续写作时自动提醒
4. **风格模仿**：上传样本，AI 学习并模仿该风格
5. **版本分支**：像 Git 一样，文字可多分支，可合并
6. **语音交互**：语音输入、朗读输出
