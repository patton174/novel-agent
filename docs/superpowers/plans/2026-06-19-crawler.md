**Goal:** 编排器 goal DB 持久化 + Redis 单例崩溃恢复 + Java 令牌桶分布式并发 + 调度策略可配（优先级/定时/重试/按站限速）+ SSE 实时推送。

**Architecture:** Java `CrawlJobService.startJob` dispatch 前 acquire Redis 令牌桶（Lua 原子），失败 429/排队；`CrawlScheduler`（@Scheduled）扫描定时 job + QUEUED 重试；编排器 goal 从 Redis 迁 DB（`crawl_orchestrator_state` 单行表），python daemon Redis SETNX 单例 + 周期续约崩溃恢复；SSE `StreamingResponseBody` 广播决策/状态/日志。

**Tech Stack:** Java 21 / Spring Boot / JPA / Flyway / Redis（StringRedisTemplate + DefaultRedisScript）/ @Scheduled；python-ai asyncio / redis；前端 EventSource。

## 关键集成事实（来自 codebase 勘察）

1. **CRM 编排器端点已存在**：`CrmCatalogController` `/api/content/crm/crawl/orchestrator*`（GET/PUT goal、wake、clear、decisions）+ `CrmCrawlBiz` + `CrawlOrchestratorStateService`（**Redis** 存 state `crawl:orchestrator:state` + decisions list）。python `OrchestratorClient.get_state()` 已调 `GET /internal/crawl/orchestrator`。本计划把 state 从 Redis 迁 **DB**（`crawl_orchestrator_state` 表），Redis 降为决策日志缓冲。
2. **无 SseEmitter / 无 Lua 先例**：令牌桶用 `DefaultRedisScript`（首例）；SSE 用 `ResponseEntity<StreamingResponseBody>` + `MediaType.TEXT_EVENT_STREAM_VALUE`（仿 `AgentStreamController`）。
3. **AuthRoleSupport 不在盘上**：T1 创建（模块3/5 共用，谁先实现谁建）。
4. **CrawlJobEntity** 15 列，加 priority/max_retries/retry_count/schedule_cron/next_run_at 干净。`startJob`（:88）发 `CrawlDispatchMessage(jobId, sourceUrl, configJson, 1)`。
5. **@EnableScheduling** 已在 `NovelStudioApplication:34`；`@Scheduled` 先例 `SiteSettingsBiz:59`、`RunProxySchedulingConfig:16`。
6. **SiteSettingsBiz** 无 typed `getInt`，需加 `getInt(key, default)` 读 `crawl.max_concurrent_jobs`（默认 2，令牌桶容量）。
7. **python daemon** `_wake_event` + poll 已有（loop.py:125）；`start_orchestrator_background`（main.py:61）按 `crawl_orchestrator_enabled` 启动。需加 Redis SETNX 单例锁。
8. **redis_client.py**（模块5 计划建，python Redis 单例）— 若模块5 未实现则本计划 T13 先建。
