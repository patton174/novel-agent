**Goal:** 书库书解析后自动索引到 Milvus（私人 library:/公共 catalog: 命名空间）+ LLM 摘要；聊天区 @引用注入参考书目（user_message 不动，独立 system 区块）；agent SearchKnowledge 扩展 scope=book: 检索该书片段。

**Architecture:** 模块5 解析回写 catalog 后发 MQ `LIBRARY_INDEX`（私人书）→ worker 逐章 POST python `/api/rag/index/chapter`(novel_id=library:<uid>:<id>) + python `/internal/library/summarize` 生成摘要回写 description。前端 @picker 选书 → AgentStreamRequest 带 referencedBooks → Java 组装(摘要+章标题)注入 context.referenced_books → python system prompt 独立区块渲染 → agent SearchKnowledge(scope=book:) 按需检索。

**Tech Stack:** Java 21 / Spring Boot / JPA / Flyway / RabbitMQ / RestClient；python-ai FastAPI / pydantic / httpx；前端 React+TS / secureFetch。

## 关键集成事实（来自 codebase 勘察）

1. **模块5 catalog 字段尚未落地**：`CrawlCatalogNovelEntity` 无 owner_id/source/uploader_file_id（plan-only）。**本计划假设模块5 已先实现**（执行顺序 5→4）。T1 迁移在模块5 V15 之上加 V19。
2. **Milvus 复用**：`novel_chapters` collection 靠 `novel_id` 字段区分命名空间，`index_chapter(novel_id=...)` 接受任意字符串——私人 `library:<uid>:<id>` 无需新 collection/schema。
3. **CatalogIndexListener 是 LIBRARY_INDEX listener 模板**：worker 已有 `@RabbitListener(queuesToDeclare=@Queue("agent.catalog-index.queue"))` + POST `/api/rag/index/chapter`(novel_id=`catalog:<id>`)。LIBRARY_INDEX 镜像之，前缀换 `library:<uid>:`。
4. **AgentRunContextDto 4 层镜像**：record(snake_case) → python AgentRunContext(pydantic)。DTO 由 `AgentRunState.toContextDto` 构建 assembler 建 Map。加 `referenced_books` 需同步 4 处。
5. **AgentStreamRequest**（record, snake_case）+ `toGatewayStreamBody` 直传——加 `referencedBooks` 字段直通 Java。
6. **SearchKnowledge**（knowledge.py:15）仅 `search_novel(ctx.novel_id)`；`search_novel(novel_id,...)` 单命名空间。扩展加 `scope` 参数。
7. **run_context.assemble_run_context** 返回 dict（top-level novel/memory/working 等）；参考书目作新 top-level `library` 区块。
8. **前端 send**：`useEditorAgentStream.ts:878` `openAgentStream({message,...})` + `toGatewayStreamBody` 直传；`ChatComposer.tsx` textarea+send。
