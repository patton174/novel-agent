**Goal:** 审批+CDK 混合升级（无真实支付）+ 余额机制 + RPM 限流 + overage 赊账 + 月度续费 job + free 计划 seed。

**Architecture:** Java billing 模块新增 user_balance/redemption_code/redemption_record/upgrade_request 表 + overage_micros 列；QuotaBiz 加 overage 分支（超配额扣 user_balance，不足赊账）；RPM 用 billing 内 BillingRpmChecker（StringRedisTemplate 滑窗，复刻 auth RateLimitService 逻辑，因 agent 不依赖 auth 模块）；CDK 兑换/审批/余额/续费 4 个 Biz + @Scheduled 续费 job；BillingAuthController/BillingCrmController 加端点；前端 BillingPage 余额/兑换/申请 + AdminBilling 4 tab。

**Tech Stack:** Java 21 / Spring Boot / JPA / Flyway / Redis（StringRedisTemplate）/ @Scheduled；前端 React+TS / secureFetch。

## 关键集成事实（来自 codebase 勘察）

1. **`overage_policy` 列/字段已存在**（V3 + ProductPlanEntity L46 默认 block），但**未接 DTO/Biz/QuotaBiz**——本计划接通（PlanCrmUpsertReq 加字段、PlanCrmBiz.applyFields 设值、QuotaBiz 读）。
2. **RPM 跨模块约束**：`RateLimitService` 在 auth 模块，agent 不依赖 auth（且 billing 不能依赖 auth——cycle）。**方案**：billing 内新建 `BillingRpmChecker`（StringRedisTemplate 滑窗，复刻 RateLimitService.check 逻辑），由 `QuotaBiz.checkAndReserveRun` 调用（agent 经 BillingInternalClient in-process 调 QuotaBiz）。`TooManyRequestsException` 在 studio-kernel（agent/billing 均可用）。
3. **free 计划 bug**：`SubscriptionBiz.ensureDefaultSubscription`(:32) 调 `createSubscription("free")` → `requireActivePlanByCode("free")` 抛（V4 无 free seed）——本计划 V11 seed free 修复。
4. **UsageReportBiz.upsertPeriodSummary**(:83) 是 overage 扣余额 + overage_micros 累加 hook 点；`incrementRedis`(:109) key `billing:usage:<uid>:<period>:tokens/runs`。
5. **QuotaBiz.checkAndReserveRun**(:28) 读 Redis tokensUsed/runsUsed，超抛 BILLING_QUOTA_EXCEEDED（block 路径）；overage 分支加在此 `if(!allowed)` 处。
6. **SubscriptionBiz.changeUserPlan**(:70) 是 CDK plan 兑换 + 审批 plan 批准的统一 hook（含 audit + invalidatePlanCache）。
7. **BillingAuthController** 已存在（`/api/billing/auth/*`），余额/兑换/申请端点加此；**BillingCrmController** 已存在，CDK/审批/余额端点加此。
8. **AuditLogService.log(actorId, action, targetType, targetId, before, after)** 是审计写原语（赊账结算/CDK/审批用）。
9. **前端 BillingPage** 3 卡（usage/bill/events），CTA 仅 Link 到 /pricing /contact；**PricingPage** CTA 全 Link /contact；admin 无单 AdminBillingPage（PlansPage/RevenuePage/UserEditDialog/AuditLogPage 分散）——本计划新建 AdminBillingPage 聚合 CDK/审批/余额/赊账 4 tab。
10. **UsagePeriodSummaryEntity** 复合主键 `(userId, periodYyyyMm)`，加 overageMicros 字段。
